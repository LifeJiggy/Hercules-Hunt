# Rules Module

Always-active behavioral guardrails loaded at session start. Each rule file constrains AI behavior in a specific domain.

| File | Description |
|------|-------------|
| `api-testing.md` | Rules for API security testing methodology |
| `auth-testing.md` | Rules for authentication testing methodology |
| `chain-rules.md` | Rules for exploit chain construction and validation |
| `evidence.md` | Rules for evidence capture and handling |
| `hunting.md` | Rules for general vulnerability hunting |
| `js-analysis.md` | Rules for JavaScript bundle analysis |
| `js-deobfuscation.md` | Rules for JavaScript deobfuscation |
| `mindset.md` | Rules for operator mindset and discipline |
| `mobile-testing.md` | Rules for mobile application testing |
| `recon.md` | Rules for reconnaissance methodology |
| `reporting.md` | Rules for report writing and submission |
| `scope.md` | Rules for scope boundary enforcement |
| `windows-workflow.md` | Rules for Windows-native workflow |
